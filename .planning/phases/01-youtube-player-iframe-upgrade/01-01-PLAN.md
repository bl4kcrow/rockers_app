    Use the compiler-guided approach (per D-02): run `flutter analyze`, fix the reported errors, repeat until zero errors. Apply changes to all 5 files in a single batch pass (per D-01). The following changes are required:

    **1. `video_playback_screen.dart` — Main migration target:**
    - **Remove** `YoutubePlayerScaffold` entirely (per D-05) — the wrapping `YoutubePlayerScaffold(controller: ..., autoFullScreen: ..., defaultOrientations: ..., builder: (context, videoPlayer) { ... })` must be replaced.
    - **Remove** `autoFullScreen: true` — v6 handles this internally (per DEP-03).
    - **Remove** `defaultOrientations: [...]` — removed in v6 (per DEP-03).
    - **Remove** `setFullScreenListener` call in `initState()` — v6 handles fullscreen lifecycle internally (per DEP-03). Remove the `SystemChrome.setEnabledSystemUIMode` call that was paired with the listener, since the scaffold will no longer be popping fullscreen to a portrait-only state.
    - **Keep** the `StreamSubscription<YoutubePlayerValue>` listener for `PlayerState.ended` → `next()` — this event-based pattern is v6-compatible.
    - **Keep** `PopScope` for provider invalidation — no change needed.
    - **Keep** the `_PlayerScaffoldBody` layout structure using `OrientationBuilder` (per D-06) — the portrait/landscape split remains.
    - **Replace** the builder pattern: instead of receiving `videoPlayer` (Widget) from `YoutubePlayerScaffold.builder`, create `YoutubePlayer(controller: playerProvider.playerController)` directly as a widget and pass it to `_PlayerScaffoldBody` as the `player` argument.
    - **Place** `YoutubePlayer` inside `SafeArea` (per D-07). The current structure wraps `YoutubePlayerScaffold` in `SafeArea`; after migration, `SafeArea` wraps `Scaffold` (which wraps `_PlayerScaffoldBody`).
    - Use auto aspect ratio (per D-08) — do not set `aspectRatio` on `YoutubePlayer` so v6 uses the iframe's intrinsic dimensions.

    **2. `video_player_provider.dart` — Verify controller params:**
    - Keep `YoutubePlayerController(params: YoutubePlayerParams(...))` — this constructor signature remains in v6.
    - Keep `enableCaption: false`, `mute: false`, `showFullscreenButton: true`, `strictRelatedVideos: true`, `showVideoAnnotations: false` — these are v6-compatible.
    - Keep `toggleFullScreen()` — this method exists in v6 (per DEP-05).
    - Keep `cueVideoById`, `loadVideoById`, `playVideo`, `pauseVideo` — these are v6-compatible.
    - Keep `_playerController.close()` in `dispose()` — v6-compatible.
    - If the compiler reports any removed/renamed params, remove them and let the compiler settle.

    **3. `video_controls.dart` — Verify YoutubeValueBuilder:**
    - `YoutubeValueBuilder(controller: ..., builder: (context, value) => ...)` works in v6 (per DEP-05).
    - The `value.playerState` checks for `PlayerState.paused`, `PlayerState.cued`, `PlayerState.playing` remain valid.
    - No code changes expected here unless the compiler flags something.

    **4. `thumbnail_image.dart` — Verify getThumbnail:**
    - `YoutubePlayerController.getThumbnail(videoId: videoId)` and `YoutubePlayerController.getThumbnail(quality: ThumbnailQuality.high, videoId: videoId)` are v6-compatible (per DEP-05).
    - No code changes expected here.

    **5. `song.dart` — Verify convertUrlToId:**
    - `YoutubePlayerController.convertUrlToId(videoUrl)` is v6-compatible (per DEP-05).
    - No code changes expected here.

    **Iteration loop:** After each round of edits, run `flutter analyze`. Fix any remaining errors. Repeat until `flutter analyze` reports 0 errors (per DEP-06). The compiler is the source of truth for what needs to change — let it guide each fix.
